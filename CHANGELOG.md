# Changelog

Все значимые изменения в этом проекте документируются здесь.
Формат: [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версионирование: [SemVer](https://semver.org/lang/ru/).

## [1.3.0] — 2026-07-12

Релиз стабилизации: полный аудит кода, безопасности и зависимостей.

### Добавлено
- Полноэкранный график на Android: разметка осей (значения, заголовок,
  подпись оси X, маркер последнего значения) и альбомная ориентация —
  разворачивается весь контент, включая кнопку, касания работают верно.
- Вкладка «Сеть» подтягивает **текущие настройки модема**: отмеченные
  бэнды Band Lock и выбранный режим антенны подставляются прямо в
  элементы управления. Только GET-запросы — существующая фиксация на
  модеме не затрагивается. Если модель не отдаёт endpoint, показывается
  предупреждение (пустые чекбоксы иначе выглядели бы как «фиксации нет»).
- Модуляция DL и UL (MCS с расшифровкой в QPSK/16/64/256-QAM), мощность
  передатчика, режим MIMO, TAC, ширина канала UL, месячный трафик —
  на обеих платформах. Поля опциональны: показываются, если роутер их
  отдаёт.
- Логирование в Android-версии (видно в `adb logcat`) — раньше ошибки
  на устройстве терялись молча.
- Тесты: проверка, что все имена, импортируемые из `core`, существуют
  (ловит рассинхрон до сборки APK), и что `on_pre_enter` не обращается
  к KV-виджетам напрямую. Всего 89 тестов.

### Изменено
- Расширен набор правил линтера: добавлены `UP` (современный синтаксис),
  `SIM` (упрощения), `C4` (генераторы). Код приведён к ним полностью.
- Типизация модернизирована под Python 3.10+ (`dict` вместо `Dict`,
  `X | None` вместо `Optional[X]`).
- PyInstaller обновлён до 6.21+ (был 6.10).
- Сборочный стек Android (buildozer 1.5.0, python-for-android
  v2024.01.21, NDK 25b, cython 0.29.36) намеренно оставлен закреплённым:
  свежие версии тянут Python 3.14, где ломается `pyjnius`.

### Исправлено
- Краш Android на старте: `on_pre_enter` первого экрана вызывается до
  привязки KV-виджетов — обращение к ним теперь через `ids.get()`.
- Зависший статус «Подключение...» на экране подключения после отключения.
- Двойное нажатие «Подключиться» на Android больше не поднимает второй
  воркер-поток.
- Тихие `except: pass` заменены на логирование — ошибки больше не
  теряются.
- Толщина линии графика ограничена сверху (в полноэкранном режиме была
  слишком жирной).

### Безопасность
- Аудит: `eval`/`exec`/`pickle`/`subprocess` не используются;
  захардкоженных секретов нет; пароль роутера живёт только в оперативной
  памяти и никогда не пишется на диск.

## [1.2.0] — 2026-XX-XX

Крупный релиз: вторая платформа (Android), выделение общего ядра в
пакет `core/`, локализация RU/EN.

### Добавлено
- **Android-версия** (Kivy + Buildozer) с полным функционалом монитора:
  RSRP/RSSI/SINR/RSRQ с пиками и цветовой индикацией, стрелка тенденции,
  джиттер, графики с полноэкранным режимом и выбором параметра, экран
  «Инфо» (вышка, SIM/устройство, состояние), экран «Сеть» (Band Lock,
  переключение антенн, перезагрузка, тест белых списков), подсказка по
  ошибкам подключения, тестовый режим с демо-данными для эмулятора.
  Сборка APK через GitHub Actions.
- **Двуязычный интерфейс** (русский / английский) с переключением «на
  лету»; вся текстовая часть вынесена в `core/i18n.py`.
- Общий пакет **`core/`** (constants, parsers, signal_analysis,
  whitelist, i18n), переиспользуемый Windows- и Android-версиями.
- Индикатор тенденции RSRP (↑ / → / ↓) с подсказкой для монтажника.
- Авто-переподключение к роутеру при обрыве с экспоненциальным backoff.
- Экспорт сессии измерений в CSV для отчётов клиенту (Windows).
- CLI-аргументы `--ip`, `--password`, `--verbose`, `--version`.
- Расшифровка LTE-бандов из любых форматов API (`'7'`, `'LTE BAND 20'`,
  `'B3+B7'`, битмаски `0x40`) в человекочитаемый вид с указанием частоты.
- Поле EARFCN и fallback-определение band по EARFCN.
- Блок «SIM / Устройство»: IMEI, IMSI, ICCID, номер, серийный номер,
  модель, прошивка.
- Кнопка перезагрузки роутера.
- Тест «белых списков (РФ)» через TCP-пробы на `:443`.
- Аудио-помощник и «крышный режим» (Windows).
- Расширенный список операторов (PLMN_MAP) для РФ, РБ, KZ, UA.
- Band Lock для девяти диапазонов РФ/СНГ: B1, B3, B5, B7, B8, B20, B38,
  B40, B41.
- Тесты на pytest (`tests/test_core.py`), линтер ruff, CI lint+test+build
  для обеих платформ.

### Изменено
- График переписан с matplotlib на голый `tk.Canvas` (Windows) и Kivy
  canvas (Android) — без тяжёлых зависимостей.
- Монолитный `main.py` разделён: вся логика в `core/`, в `main.py`
  остался только UI-слой Tkinter.
- Экран мониторинга Android переведён на пропорциональную раскладку
  (`size_hint`) — адаптируется под телефоны и планшеты без прокрутки.
- Парсинг `cell_id` отбрасывает заведомо невалидные значения.
- `extract_number` использует строгий `re.fullmatch`.

### Исправлено
- Потокобезопасность: фоновый поток мониторинга не падает при
  обнулении клиента во время `disconnect()`; смена экрана из фонового
  потока на Android — только через главный поток.
- Совместимость крипто-зависимости на Android: `pycryptodomex`
  (нет рецепта p4a) заменён на `pycryptodome` с перенаправлением
  неймспейса `Cryptodome` → `Crypto`.
- Корректный `logout()` при отключении.
- Полная локализация RU/EN: метки уровней сигнала, бэнды, режимы антенн,
  вердикты белых списков, подписи графиков.

### Удалено
- Сохранение пароля и IP на диск — программа портативна.
- Зависимости `matplotlib` и `requests`.
